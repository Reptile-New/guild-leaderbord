BBoards des Guildes

Règlement & soumission

Principe simple : tout le monde peut jouer, tout est public, et la preuve l'emporte sur la confiance. Les boards vivent sur GitHub — chaque record est un commit, chaque litige est une issue, tout l'historique est auditable.

1. Vérification : deux niveaux de confiance

Nous avons choisi un système hybride plutôt que d'imposer la vidéo à tous ou de croire tout le monde sur parole :

  • ✔ Vérifié — la soumission inclut une vidéo (ou VOD non coupée) relue par un mainteneur des Boards : combat visible en entier, chrono vérifiable, effectif comptable. C'est l'étalon-or.
  • ✎ Sur l'honneur — le maître de guilde déclare le résultat sans preuve. L'entrée est listée et classée, mais marquée visuellement, et…
La règle d'or : sur chaque tableau, les entrées vérifiées passent toujours devant les déclarées — même avec un temps plus lent. Et la première place ne porte le titre « Record du serveur » que si elle est vérifiée. Pas de preuve, pas de record. Déclarer sur l'honneur te place quand même au tableau et garde le processus léger ; filmer te donne la couronne.

Pourquoi ce compromis ? Le tout-vidéo alourdit et décourage la participation détente ; le tout-confiance tue la crédibilité. Ainsi, une guilde tranquille poste son temps en 2 minutes, et une guilde compétitive a une vraie raison de filmer ses kills.

2. Comment soumettre

Tout passe par le dépôt GitHub — aucun compte sur un site tiers, aucun serveur à maintenir :

  • Inscrire sa guilde : ouvrir l'issue « Inscription de guilde » (nom, tag, région, effectif, niveau moyen, XP totale). Un mainteneur l'intègre dans data/guilds.json.
  • Soumettre un record PvE : issue « Record PvE » — boss, difficulté, temps de kill, effectif, date, lien vidéo si disponible. Ou une PR directement sur data/pve-records.json.
  • Cote de ladder : même logique, vers data/pvp.json.
  • Résultats de tournoi : l'organisateur soumet l'arbre complet vers data/tournaments.json.

Seuls les maîtres de guilde (ou un officier désigné dans l'issue d'inscription) font foi pour les soumissions de leur guilde.

3. Tableaux PvE

  • Un tableau par boss × difficulté (normal / héroïque), aligné sur le contenu du jeu : le raid Nythraxis (10 joueurs), les quatre donjons (5 joueurs), les gouffres (2 joueurs, temps sur le gouffre complet) et le boss mondial Thunzharr.
  • Le chrono court du pull à la mort du boss (de l'entrée au clear pour les gouffres).
  • Chaque guilde apparaît une fois par tableau avec son meilleur essai. Les dates de premier kill sont aussi suivies (course à la progression).
  • Barème par tableau pour le classement général (barème F1) : 25, 18, 15, 12, 10, 8, 6, 4, 2, 1.
  • Quand une MAJ du jeu change significativement une rencontre, les tableaux concernés sont archivés et repartent à zéro (la version du jeu est stockée sur chaque record).

4. Circuit PvP & tournois

  • Formats gérés : élimination directe, double élimination, poules (round robin) et système suisse.
  • Points du circuit par placement — tournois majeurs : 100 / 60 / 45 / 35 / 20 (5e–8e) ; standards : 50 / 30 / 22 / 18 / 10.
  • Chaque guilde peut organiser un tournoi standard ; les majeurs sont organisés ou co-signés par les mainteneurs des Boards.
  • Les cotes de ladder en jeu (1v1/2v2) sont affichées pour la gloire mais n'alimentent jamais le circuit — seule la compétition inter-guildes compte.

5. Contestations

  • Tout résultat peut être contesté sous 7 jours via l'issue « Contestation », avec explication.
  • Une entrée sur l'honneur contestée doit produire une preuve, sinon elle est retirée. Une entrée vérifiée contestée est re-relue par deux mainteneurs.
  • Triche avérée (vidéo montée, résultat fabriqué) = guilde suspendue des boards pour une saison.

6. Pourquoi GitHub, et le chemin vers l'« officiel »

Tout le site est statique et gratuit à héberger (GitHub Pages). Il n'y a pas de base de données : les fichiers JSON du dépôt sont la base de données, et l'historique git est le journal d'audit. Le projet est donc trivial à transférer : levy-street peut forker ou adopter le dépôt, brancher les vraies données du jeu (ou une API) sur les mêmes contrats JSON, et ces boards deviennent officiels sans rien réécrire.

Feuille de route si adoption : temps de kill poussés automatiquement par le serveur du jeu (plus besoin de vidéo), identité de guilde via OAuth, saisons avec récompenses, et tableaux de speedrun par classe.
